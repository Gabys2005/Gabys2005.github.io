Basic usage

The module has 3 functions - new(), DestroyAllGifsIn() and GetById()

The new function takes a table with options. In the most basic case the options are:

  • Screen (or Screens) = a frame/texture (or a table of them) where the gif should be1
  • Image = ID of a spritesheet2
  • FPS = frames per second (up to 60)3, basically speed of the GIF
  • FramesPerRow = how many frames are in 1 row,
  • Frames = total number of frames

1 screen needs to point to a GUI Object (ScreenGui, SurfaceGui, Frame, ScrollingFrame, TextLabel, etc.) or a Texture (which adds support to meshes/unions/other shapes of parts)
2 The Image needs to be a spritesheet. Click here to convert your GIF into a sprite sheet
3 The GIF may be slower depending on the server lag created in the game
Note: in the example below the module is in ServerScriptService 

	local GIFPlayer = require(game.ServerScriptService.GIFPlayer)

local gif = GIFPlayer.new({
	Screen = workspace.Screen.SurfaceGui,
	Image = "rbxassetid://3528512681",
	FPS = 24,
	FramesPerRow = 6,
	Frames = 26
})

gif:Play()

All functions

gif:Play() - plays the GIF

gif:Pause() - pauses the GIF

gif:Reset() - resets the GIF (puts it on the 1st frame)

gif:Destroy() - removes the GIF

Stopping all GIFs in an instance

DestroyAllGifsIn(Instance) is a function that runs gif:Destroy() on all GIFs that are a descendant of a location put as a variable 

	local GIFPlayer = require(game.ServerScriptService.GIFPlayer)

GIFPlayer.DestroyAllGifsIn(workspace.Prop)

IDs

When creating a GIF you can specify an ID for it

					GIFPlayer.new({
	ID = "123",
	...
})

When a GIF with an ID is created, any previous GIFs with the same ID will get automatically destroyed

Additionally, you can use .GetById() to get a GIF object with a specified ID. For example to pause a GIF with an id of 123 you can do: 

local gif = GIFPlayer.GetById("123")
if gif then
	gif:Pause()
end

Playing the same GIF on multiple screens

The code below shows how to play the same GIF on multiple screens without running .new on each of them 

	local GIFPlayer = require(game.ServerScriptService.GIFPlayer)
            
local surfaces = {workspace.NormalScreen1.SurfaceGui,
		  workspace.MeshScreen1.Texture}
					
local gif = GIFPlayer.new({
	Screens = surfaces,	
	Image = "rbxassetid://3528512681",
	FPS = 24,
	FramesPerRow = 6,
	Frames = 26
})
		
gif:Play()

Using multiple spritesheets

The code below shows how to use multiple spritesheets in the same GIF (mainly for better quality)

Note: you can add more than 2 sheets

local GIFPlayer = require(game.ServerScriptService.GIFPlayer)

local sheet1 = {
	Image = "rbxassetid://5138441857",		
	FPS = 2,	
	FramesPerRow = 4,
	Frames = 20
}
		
local sheet2 = {
	Image = "rbxassetid://3528512681",	
	FPS = 24,
	FramesPerRow = 6,
	Frames = 26
}

local gif = GIFPlayer.new({
	Screen = workspace.Screen.SurfaceGui.GIF,
	Mode = "Advanced",
	SpriteSheets = {sheet1, sheet2}
})	

gif:Play()

Effects

You can set the Effect setting to set up an effect when a GIF is being shown and destroyed. Effects exist mainly for switching between GIFs on the same screen.

Currently only a Fade effect is available, you can configure it with the optional Time and Blocking settings. 

local GIFPlayer = require(game.ServerScriptService.GIFPlayer)

local gif = GIFPlayer.new({
	Screen = workspace.Screen.SurfaceGui,
	Image = "rbxassetid://3528512681",
	FPS = 24,
	FramesPerRow = 6,
	Frames = 26,
	Effect = {
		Name = "Fade",
		-- It'll take 0.25 seconds to fade between GIFs
		Time = 0.25,
		-- When this setting is set to true,
		-- the code will yield when destroying a GIF
		Blocking = false
	}
})

gif:Play()
